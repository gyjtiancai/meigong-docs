前端规范方案

为什么需要规范

正所谓没有规矩，不成方圆。

团队的规范不意味着是最标准的，更多的是团队成员能够在开发上达成一致，从而提高代码整体的可读性、可维护性、可复用性、可移植性和可靠性，从根本上降低开发成本，这才是制定规范的真正意义。

命名规范

命名方法：camelCase(驼峰 - 类和构造函数除外)和PascalCase(大骆驼 - 类和构造函数)

//bad
class person {
    constructor(options) {
        this.name = options.name;
        this.age = options.age;
    }
    ap() {
        console.log('add person')
    }
}
const p = new person({
    name:'geyujie',
    age:26
})

//good
class Person {
    constructor(options) {
        this.name = options.name;
        this.age = options.age;
    }
    addPerson() {
        console.log('add person')
    }
}
const person = new Person({
    name:'geyujie',
    age:26
})

HTML规范

标签语义化或者className语义化(多个单词用 - 拼接)

<!-- bad -->
<div>头部</div>
<div>
    <div> 导航1 </div>
    <div> 导航2 </div>
    <div> 导航3 </div>
</div>
<div>侧栏</div>
<div>文章</div>
<div>页脚</div>

<!-- good -->
<header>头部</header>
<nav>
    <a href="javascript:return false"> 导航1 </a>
    <a href="javascript:return false"> 导航2 </a>
    <a href="javascript:return false"> 导航3 </a>
</nav>
<aside>侧栏</aside>
<article>文章</article>
<footer>页脚</footer>

<!-- good -->
<div class="header">头部</div>
<div class="nav">
    <div class="nav-option"> 导航1 </div>
    <div class="nav-option"> 导航2 </div>
    <div class="nav-option"> 导航3 </div>
</div>
<div class="aside">侧栏</div>
<div class="article">文章</div>
<div class="footer">页脚</div>

JavaScript规范

语法 - Es6 如：不使用var（函数级作用域），使用const 和 let（块级作用域）

//bad
var num = 1;

//good
let num = 1;
const num = 1;

注释

注释一般包括文件顶部的总述，DOM的节点描述，每个函数或者变量的说明，举个例子：

/**
 * 头部栏组件
 */
<script setup lang="ts">
import { ref } from 'vue'
import { ArrowDown } from '@element-plus/icons-vue'
import { ElMessageBox, ElMessage } from 'element-plus'
import { useRouter } from 'vue-router'
import { useUserStore } from '@/store'
import ReplacePhone from './headerComponents/ReplacePhone.vue'
import ResetPassword from './headerComponents/ResetPassword.vue'
const router = useRouter()
const userStore = useUserStore()
const [replacePhoneVisible, resetPasswordVisible] = [ref(false), ref(false)]
//退出登录
const logout = ():void => {

}
//更换手机号
const replacePhone = (newPhone: string):void => {

}
//重置密码
const resetPassword = (oldPassword: string, newPassword: string):void => {

}
</script>

<template>
    <div class="main-header">
        <!-- 下拉菜单 -->
        <el-dropdown trigger="click">
            <span class="el-dropdown-link">
                <div class="dropdown-entry">
                    <div class="header-user">
                        <img class="user-avatar"
                            src="https://cube.elemecdn.com/3/7c/3ea6beec64369c2642b92c6726f1epng.png" />
                        <div class="user-name">{{ userStore.user?.name }}</div>
                    </div>
                    <el-icon><arrow-down /></el-icon>
                </div>
            </span>
            <template #dropdown>
                <el-dropdown-menu>
                    <el-dropdown-item @click="replacePhoneVisible = true">更换手机号</el-dropdown-item>
                    <el-dropdown-item @click="resetPasswordVisible = true">修改密码</el-dropdown-item>
                    <el-dropdown-item @click="logout">退出登录</el-dropdown-item>
                </el-dropdown-menu>
            </template>
        </el-dropdown>
    </div>
    <!-- 更换手机号组件 -->
    <ReplacePhone :visible="replacePhoneVisible" @cancel="replacePhoneVisible = false" @confirm="replacePhone">
    </ReplacePhone>
    <!-- 修改密码组件 -->
    <ResetPassword :visible="resetPasswordVisible" @cancel="resetPasswordVisible = false" @confirm="resetPassword">
    </ResetPassword>
</template>
